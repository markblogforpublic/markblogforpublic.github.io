AILTES: AI English Speaking Evaluation System

An LLM-powered IELTS speaking practice platform with multi-dimensional scoring, explainable feedback — and the story of how it was built.

AILTES：基于大语言模型的英语口语评估系统

基于大语言模型的雅思口语练习平台——多维度评分、可解释性反馈，以及背后的开发故事。

calendar_today 2026-05-05 schedule ~18 min read

Abstract

This research presents AILTES, an AI-driven interactive English speaking system that uses large language models to simulate IELTS speaking exams, evaluate expression quality across multiple dimensions (fluency, lexical resource, naturalness), and provide explainable feedback. The system features four core modules — dialogue generation, expression evaluation, feedback generation, and user analytics — and supports both cloud API and local model deployment. This report also covers the development journey: technical decisions, challenges faced, and lessons learned.

摘要

本研究提出了 AILTES，一个基于大语言模型的英语口语交互系统。该系统模拟雅思口语考试场景，从流畅度、词汇丰富度、表达自然度三个维度评估表达质量，并提供可解释性反馈。系统包含对话生成、表达评估、反馈生成和用户数据分析四个核心模块，支持云端 API 和本地模型两种部署方式。本文还记录了开发历程：技术决策、遇到的挑战和心得收获。

Introduction研究背景

Large Language Models (LLMs) have made remarkable progress in NLP, but existing English learning tools still have significant gaps: they focus on writing, lack multi-turn conversation, cannot simulate real exam scenarios, and provide feedback without explainability. This research bridges that gap.

Objective: design an English speaking system with realistic interaction and explainable expression quality evaluation.

大语言模型在 NLP 领域取得了显著进展，但现有英语学习工具仍存在明显不足：侧重写作、缺乏多轮对话、无法模拟真实考试、反馈缺乏解释性。本研究旨在填补这一空白。

目标：设计一个具备真实交互能力和可解释表达质量评估的英语口语系统。

Methodology & Implementation方法与实现

Prompt Engineering

Carefully designed system prompts with IELTS rubric examples, forcing structured JSON output, and separating evaluation from dialogue to avoid role conflict.

精心设计的系统提示词，包含雅思评分标准示例，强制结构化 JSON 输出，将评估与对话任务分离以避免角色冲突。

Rule + LLM Hybrid

Rules handle basic grammar detection. LLM handles semantic understanding and naturalness — combining both approaches' strengths.

规则方法处理基础语法检测，LLM 处理语义理解与自然度评估，结合两者优势。

Why I Built This为什么做这个项目

IELTS speaking practice is a real pain point — one-on-one tutoring is expensive, and most tools either lack interactivity or provide generic feedback. I wanted to build a "virtual examiner" available 24/7 that provides detailed, actionable feedback while respecting privacy by running locally. What started as a simple question — "Can an LLM reliably evaluate spoken English?" — became a full-stack application with four interconnected modules, GPU backend management, and a real-time analytics dashboard.

雅思口语练习是一个真实痛点——一对一辅导昂贵，大多工具缺乏交互性或反馈笼统。我想做一个 24/7 在线的"虚拟考官"，提供详细反馈，同时通过本地运行保护隐私。从一个简单问题开始——"大语言模型能可靠评估英语口语吗？"——最终演变为一个全栈应用。

build

Technical Decisions技术决策

GPU Backend Hot-Swapping

The trickiest part. llama-cpp-python needs different DLLs for CUDA vs Vulkan vs CPU. I built a launcher that detects your GPU, copies the right DLLs at startup, and lets you switch instantly — no Docker, no complex setup.

最棘手的部分。我写了一个启动器：自动检测 GPU，启动时复制对应 DLL，无需 Docker 无需复杂配置。

Scoring Prompt Engineering

Getting consistent 1-9 scores required forcing structured JSON output, providing IELTS rubric examples in the system prompt, and separating evaluation from dialogue to avoid role conflict.

强制结构化 JSON 输出、在 system prompt 中嵌入雅思评分标准、将评估与对话任务分离。

Bilingual by Design

Full English and Chinese support in both the frontend UI and the backend evaluator prompts — following the same pattern as this blog, making the project accessible to a wider audience.

沿用博客的双语模式——前端 UI 和后端评估 Prompt 都完整支持中英文。

Challenges Faced遇到的挑战

gpu

DLL Hell

Managing CUDA/Vulkan DLLs across GPU vendors was a nightmare. Solution: pre-bundle backends, let users download only what they need, hot-swap at launch.

管理不同 GPU 厂商的 DLL 非常痛苦。预打包所有后端，用户只需下载需要的部分。

score

Scoring Consistency

LLMs are inherently inconsistent. Reliable 1-9 scoring required many iterations of prompt refinement and testing.

LLM 天然不一致。获得可靠评分需要多次迭代优化 Prompt。

bolt

Local Model Performance

GGUF models are slow (30s+ per response). The UI needed clear loading states and expectation-setting banners.

本地 GGUF 模型推理慢。UI 需要清晰的加载状态和预期提示。

What I Learned收获与成长

  • System design matters. Four modules with clear separation of concerns made the project scalable and maintainable.
  • LLMs are tools, not magic. A well-prompted small model can outperform a poorly-prompted large one on specific tasks.
  • GPU programming is still hard. Pre-bundling backends with clear setup instructions is essential for user adoption.
  • Education is high-stakes. Telling someone their English is a "5" when it's a "6" affects confidence. Reliable evaluation requires humility and testing.
  • Choose the right license. GPL (Non-Commercial) for educational use, MIT for broader use — each project needs the right fit.
  • 系统设计至关重要。四个模块、清晰的关注点分离，让项目可扩展可维护。
  • LLM 是工具，不是魔法。Prompt 好的小模型在特定任务上可能胜过 Prompt 差的大模型。
  • GPU 编程依然困难。预打包后端和清晰指南对用户采用至关重要。
  • 教育是高风险的。评分错误会影响信心。可靠评估需要谦逊和持续测试。
  • 选择合适的许可证。教育用途 GPL，广泛用途 MIT——每个项目需要匹配目标的许可证。
