Phos v1.0

Local small model toolkit with llama-cpp-python + Streamlit GUI, fully offline operation

Phos v1.0 — Feature Report

Project Overview

Phos is a local small model toolkit based on llama-cpp-python, providing a visual interface through Streamlit GUI. Users can load local GGUF format models to perform translation, summarization, chat, classification, extraction, visual understanding, speech synthesis, and more — all running locally without an internet connection.

System Architecture

minimalLLMagent/
├── app.py               # Streamlit GUI entry point (sidebar + 7 tabs)
├── requirements.txt     # Dependency declaration
├── phos/
│   ├── __init__.py      # Package exports

Core Module Details

1. phos/model.py — Model Management Core

ModelManager is the central class of the entire system, loading GGUF models directly via llama-cpp-python.

Key Methods:

MethodDescription
load(path, n_ctx, n_gpu_layers)Load text-only model
load_multimodal(path, mmproj, n_ctx, n_gpu_layers)Load multimodal model (with mmproj vision projector)
unload()Release model memory
complete(messages, temperature, max_tokens)Single-turn generation, returns full text
stream(messages, temperature, max_tokens)Token-by-token streaming generation, yields (text, stats)
_has_images(messages)Detects whether messages contain images

Features:

  • Auto-scans NVIDIA CUDA DLL paths at startup, supports GPU acceleration
  • Auto-detects multimodal messages during streaming generation, falls back to non-streaming mode to ensure proper image processing
  • Records performance statistics after each generation (total time, token count, tokens/sec, time to first token)

2. phos/tasks.py — Text Tasks

Five independent functions, all accepting a manager parameter:

FunctionFunctionalityInputOutput
translate(text, target_lang, source_lang)TranslationText + target languagestr
summarize(text, style, max_length)Summarization/rewritingText + style parametersstr
classify(text, categories)Text classificationText + category listdict (category, confidence, reasoning)
extract(text, fields)Information extractionText + field listdict (field name -> value)
chat(messages, system_prompt)Multi-turn conversationMessage historystr

classify/extract use regex to extract JSON from output, not relying on json_mode, compatible with all models.

3. phos/vision.py — Image Understanding

FunctionFunctionality
describe_image(image_path, prompt)Image description, supports custom prompts
ask_about_image(image_path, question)Image Q&A

Image files are read and base64-encoded, then passed to the model via image_url type messages.

4. phos/tts.py — Speech Synthesis

Based on Orpheus-3B TTS model + SNAC audio decoder:

Text -> Orpheus GGUF model -> Audio tokens -> SNAC decoder -> PCM audio -> WAV file
ComponentPurpose
synthesize(text, voice, temperature)Main entry point, returns WAV bytes
pcm_to_wav(pcm_bytes, sample_rate)PCM to WAV conversion
SNAC (24kHz)Audio decoder, auto-downloaded and cached

Supports 8 voices: tara, leah, jess, leo, dan, mia, zac, zoe

5. phos/presets.py — Parameter Presets

FunctionFunctionality
load_presets()Read ~/.phos/presets.json
save_preset(name, settings)Save n_ctx, n_gpu_layers, temperature, max_tokens
delete_preset(name)Delete preset

Main Area Tabs

Tab 1: Chat

  • Streaming output (character-by-character display)
  • Maintains multi-turn conversation history
  • Optional System prompt
  • When using multimodal model: Shows image upload interface (paperclip icon to attach images), images sent with messages
  • Performance stats bar: Total time, token count, tokens/s, time to first token

Tab 2: Translation

  • Source language/target language dropdowns (supports auto-detection)
  • Target language defaults to Chinese

Tab 3: Summarization/Rewriting

  • Style selection: concise / formal / casual / bullet-points / elaborate
  • Custom output length

Tab 4: Classification/Extraction

  • Classification mode: Input text + category list -> JSON (category, confidence, reasoning)
  • Extraction mode: Input text + field list -> JSON (field -> value)

Tab 5: Vision (shown when multimodal model is loaded)

  • Upload image + custom question
  • Returns model analysis result

Tab 6: Speech Synthesis

  • Enter text + select voice
  • Play audio + download WAV
  • Depends on snac package

Startup

cd minimalLLMagent
pip install -r requirements.txt
streamlit run app.py

Open in browser at http://localhost:8501

Technical Highlights

  1. Zero-configuration auto-detection: Auto-scans CUDA DLLs at startup, no manual environment variable setup needed
  2. GPU/CPU one-click switching: Sidebar radio toggle, n_gpu_layers auto-adjusts
  3. Multimodal auto-adaptation: After loading a model with mmproj, the chat tab automatically shows the image upload interface
  4. Streaming generation: Chat tab displays character-by-character, non-streaming for regular tasks ensures stability
  5. Performance visualization: Displays tokens/sec, TTFT, and other key metrics after each generation
  6. Preset persistence: Parameter combinations saved to ~/.phos/presets.json, preserved across restarts
  7. Native directory selector: Uses tkinter to open system-level directory selection dialog
