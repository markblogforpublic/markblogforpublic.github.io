1. Project Overview

Argus is an AI-driven desktop automation agent that enables AI models to control a computer like a human through a screenshot -> AI analysis -> action execution loop. Supports multi-vendor APIs (Anthropic official / OpenAI-compatible third-party), with built-in safety analysis, automatic input method switching, smart scrolling, and more.

3.3 Smart Input Method Switching

Language Detection (_detect_language):

  • Detects CJK characters based on Unicode ranges (CJK Unified Ideographs, Kana, Hangul, etc.)
  • Threshold: CJK character ratio > 15% determines CJK text
  • Mixed text (e.g., “Hello你好”) also triggers switching

Input Method Switching (_ensure_ime_for_text):

  • CJK text detected -> Automatically sends Shift key to switch input method
  • Applicable to Microsoft Pinyin, Sogou Input, and other input methods that use Shift to toggle Chinese/English by default
  • After switching, pastes via clipboard (Ctrl+V)

3.4 Safety Analysis System

Three-tier Security Assessment:

LevelTrigger ConditionHandling
SAFEmouse_move, screenshot, cursor_position, waitAuto-execute
CAUTIONleft_click, right_click, type, key, scroll, dragLog record, auto-execute
DANGEROUSSpecific hotkeys, dangerous commands, rapid consecutive clicksDialog confirmation (tkinter messagebox)

DANGEROUS Trigger Rules:

CategoryTrigger Condition
Dangerous hotkeysalt+f4, ctrl+alt+del, ctrl+shift+esc, delete, backspace, alt+tab, win+d, win+l, win+r, ctrl+w, ctrl+q, f4
Dangerous commandsrm , del , format , shutdown, restart, taskkill, sudo , chmod 777, rd /s, diskpart, bcdedit
Consecutive clicksSame-type click action more than 5 times

Dialog Confirmation:

  • Uses tkinter.messagebox.askyesno
  • Displays action details + risk description
  • User clicks “Yes” to execute, “No” to skip and notify AI to try alternative approaches

Two Modes:

  • safe (default) — DANGEROUS actions require confirmation
  • dangerous — All actions auto-execute

CLI Flow:

  1. Auto-detects screen resolution
  2. Auto-imports API configuration from .env + cli_config.json
  3. Shows existing profiles, select or create new
  4. Detects model capabilities (vision / computer_use / tool_use)
  5. Selects safety mode (safe / dangerous)
