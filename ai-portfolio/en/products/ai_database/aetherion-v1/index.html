Version: 0.1.0 Date: 2026-05-30 Tech Stack: Python 3.11+ · FastAPI · React 18 · Vite · TailwindCSS 4 · Anthropic SDK

1. Project Overview

Aetherion is an AI-native database operating system that transforms databases from passive storage into intelligent collaborative systems. It leverages the Anthropic Claude API to enable natural language to SQL conversion, database analysis, risk assessment, and autonomous optimization, while providing a rigorous security governance framework.

Core philosophy:

Databases should not merely store information.
They should understand, analyze, optimize, and collaborate.

3. Dual-Layer Execution Matrix

Aetherion’s core design is a dual-layer matrix of Execution Permission Layer × Cognitive Intelligence Layer, producing 9 distinct AI roles.

3.1 Execution Permission Layer (WHAT — What AI is Allowed to Do)

ModeDescriptionSecurity Level
planRead-only mode. AI can analyze schemas, generate SQL, simulate execution, and explain query plans, but cannot modify the database.Highest
allow_editHuman-AI collaboration mode. AI can execute queries, modify records, optimize tables, and create indexes, but each critical operation requires user confirmation.Medium
bypassFully autonomous execution mode. AI can autonomously optimize systems, execute migrations, and fix performance issues. The Guardian system still blocks fatal operations.Controlled

4. Core Module Details

4.1 Cognitive Engine (aetherion/ai/)

Client (client.py)

  • Wraps the Anthropic Python SDK with support for custom API URL and key
  • Prompt Caching: system prompt and schema context use ephemeral cache control
  • Streaming output: stream_message() returns text chunks in real time
  • Extended Thinking: deep reasoning enabled in Hōkiboshi mode (thinking_budget=10000)
  • Token tracking: records input/output/cache_read/cache_creation tokens and cost estimates
  • Automatic environment conflict cleanup: detects and clears leftover Claude Code session variables such as ANTHROPIC_AUTH_TOKEN

Cognitive Engine Base Class (cognitive_engine.py)

  • Abstract interface: build_reasoning_chain() and get_tools() implemented by each engine
  • Automatic routing: selects Extended Thinking / Tool Use / Simple Message mode based on thinking_budget
  • Schema context injection: passes database structure via the schema_context parameter

4.2 Database Adapters (aetherion/db/)

Abstract Base Class (base.py)

  • Unified interface: connect(), disconnect(), get_schema(), execute(), explain(), get_stats()
  • Context manager: with adapter: for automatic connect/disconnect
  • Schema to string: to_context_string() generates an AI-readable database structure description
  • Data models: ColumnInfo, IndexInfo, ForeignKeyInfo, TableInfo, SchemaInfo, QueryResult, ExecutionPlan, DatabaseStats

SQLite Adapter (sqlite_adapter.py)

  • WAL mode: enables PRAGMA journal_mode=WAL to optimize concurrent read performance
  • Foreign key constraints: PRAGMA foreign_keys=ON
  • Full schema extraction: PRAGMA table_info, PRAGMA foreign_key_list, PRAGMA index_list
  • Row count statistics: SELECT COUNT(*) for exact counts
  • EXPLAIN support: parses opcodes to calculate Seq Scan / Index Scan statistics

PostgreSQL Adapter (postgresql_adapter.py)

  • Connects via psycopg2-binary
  • information_schema queries for columns, foreign keys, indexes
  • pg_indexes / pg_class / pg_stat_activity for statistics
  • EXPLAIN JSON format output with recursive execution plan tree parsing

4.3 Security Governance Layer (aetherion/security/)

AST Validator (ast_validator.py)

All AI-generated SQL must pass AST parsing validation before execution. Uses the sqlglot library.

CheckActionDescription
DROP DATABASE / DROP SCHEMABLOCKGuardian system permanently blocks
TRUNCATE TABLEBLOCKPermanently blocked
DELETE without WHEREBLOCKCondition must be specified
UPDATE without WHEREBLOCKCondition must be specified
ALTER TABLEREQUIRE_CONFIRMRequires user confirmation
CREATE INDEXREQUIRE_CONFIRMMay impact write performance
Multi-statement injectionBLOCKDetects ;-separated multiple SQL statements
JOIN count > 4FLAGWarning for complex queries

Risk Scoring Engine (risk_engine.py)

Three-dimensional risk assessment:

  1. Risk Level: LOW → MEDIUM → HIGH → CRITICAL
  2. Impact Scope: NONE → SINGLE_ROW → MULTI_ROW → TABLE → SCHEMA → DATABASE
  3. Confidence: 0.0 ~ 1.0 (below threshold triggers sandbox execution)

Composite scoring formula: score = base_risk × 0.6 + scope_factor × 0.3 + confidence_factor × 0.1

Execution path decisions:

  • LOW → Direct execution in all modes
  • MEDIUM → Bypass executes directly, Allow Edit requires confirmation, Plan blocks
  • HIGH → Sandbox execution + confirmation
  • CRITICAL → Guardian permanently blocks

Role Registry (role_registry.py)

Pre-configured 9 roles, each containing:

  • Tool whitelist (limits tools available to that role)
  • Safety constitution type (Safety / Research / Enterprise / Speed)
  • Token budget and Thinking budget
  • Streaming output toggle

4.4 Governance Matrix (aetherion/governance/matrix.py)

Core orchestrator, processes an 8-step pipeline:

User Input → ① Parse Role → ② Fetch Schema → ③ Build Context
         → ④ Get Engine → ⑤ AI Processing → ⑥ Extract Content and Tool Calls
         → ⑦ Validate SQL (AST + Risk) → ⑧ Execute or Return

Supports runtime mode switching (/mode hokiboshi, /exec bypass), with warnings on privilege escalation.

4.5 Audit Log (aetherion/audit/logger.py)

  • Storage: SQLite append-only database
  • Recorded content: timestamp, session ID, execution/cognitive mode, role name, user input, generated SQL, risk level, execution result, token consumption
  • Query interface: filter by session_id, action, limit
  • Statistics interface: total entry count, per-action counts, token consumption summary

4.6 Configuration System (aetherion/config.py)

Priority override: CLI arguments > environment variables > configuration file (aetherion.yaml) > defaults

Supported environment variables:

  • ANTHROPIC_API_KEY — API key
  • ANTHROPIC_BASE_URL — API base URL (supports proxy/relay services)
  • AETHERION_MODEL — Default model name

5. CLI (aetherion/cli.py)

aetherion chat                    # Interactive chat (default Vivace mode)
aetherion chat --mode hokiboshi   # Deep reasoning mode
aetherion chat --mode sfumato     # Exploratory mode
aetherion analyze --db sqlite:x   # Analyze database
aetherion query "natural language question"      # Single query
aetherion status                  # Show current configuration
aetherion serve                   # Start Web dashboard API server
aetherion serve --model mimo-v2.5 # Start with specified model
aetherion serve --port 8001       # Specify port

Built-in interactive mode commands:

  • /mode <hokiboshi|sfumato|vivace> — Switch cognitive mode
  • /exec <plan|allow_edit|bypass> — Switch execution mode
  • /usage — Show token consumption statistics
  • /schema — Show database structure
  • /quit — Exit

6.3 Five Panels

① AI Chat Panel (Default Panel)

  • Real-time streaming output: Displays AI responses character by character via SSE (Server-Sent Events)
  • Markdown rendering: Supports code blocks, tables, lists, and blockquotes
  • Mode switching: Top-right dropdown to select execution mode and cognitive mode
  • SQL preview: AI-generated SQL displayed in monospace code blocks
  • Role identity: Each response shows the role name used and execution time
  • Block notification: Displays a red BLOCKED label when intercepted by the Guardian system
  • Input experience: Enter to send, Shift+Enter for newline, auto-focus

② Database Browser Panel

  • Left-side tree navigation: Tables → Columns (primary key with gold key icon, foreign key with blue icon) → Type tags
  • Row count statistics: Row count displayed to the right of each table
  • Query editor: Monospace SQL input box, Ctrl+Enter for quick execution
  • Results table: Fixed headers, hover row highlighting, NULL values shown with transparency
  • Connection management: Database URL input at the top, supports SQLite and PostgreSQL
  • Database statistics: Table count, total rows, file size

③ Security and Governance Panel

  • Current mode display: Execution mode + cognitive mode dropdown switching, shows current active role name and description
  • SQL validator: One-click validation after pasting SQL, displays AST analysis results
    • Statement type (SELECT/INSERT/UPDATE/DELETE/DDL)
    • Table count, JOIN count, WHERE clause presence
    • Violation list (with severity icons)
  • Risk assessment: Risk level (with color label), score, impact scope, confidence
    • Shows lock icon and block reason when blocked
  • Role matrix: Displays all 9 roles, current active role highlighted

④ Monitoring and Audit Panel

  • Statistics cards: Total entries, input tokens, output tokens, action type count
  • Action distribution: Counts for allowed / blocked / confirmed
  • Audit log table: Time, mode, role, input, SQL, risk, action, duration, token consumption
    • Fixed table header, hover row highlighting
    • Time displayed with monospace font alignment

⑤ Settings Panel (Modal Window)

  • API Key: Password input with show/hide toggle, displays currently masked key
  • Base URL: Supports custom proxy/relay addresses (e.g. https://token-plan-cn.xiaomimimo.com/anthropic)
  • Model: Supports custom model names (e.g. mimo-v2.5, claude-sonnet-4-20250514)
  • Test connection: One-click API connectivity test, displays connection result and model name
  • Persistent save: Configuration written to aetherion.yaml file and environment variables set simultaneously
  • Configuration path hint: Displays the full path of the current configuration file

6.4 Sidebar Navigation

  • 56px narrow bar, expands to 180px on hover, 200ms transition
  • Four panel navigation icons + text labels
  • Bottom: Settings gear icon + language switch (CN/EN)
  • Current panel highlighted with brand color semi-transparent background

6.5 Internationalization (i18n)
