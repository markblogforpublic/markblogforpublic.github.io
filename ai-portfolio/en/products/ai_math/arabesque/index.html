Project Overview

Arabesque is an interactive formula visualization platform. Users input natural language or LaTeX math formulas, AI automatically parses them and generates structured visualization configurations, and the frontend renders 3D surfaces, parametric curves, vector fields, particle animations, and other graphics in real time via Plotly.js, while providing Chinese-language explanations.

Tech Stack: React + TypeScript + Vite (Frontend) | Python FastAPI (Backend) | Anthropic SDK + OpenAI SDK (AI) | Plotly.js (Visualization)

Feature Modules

1. Formula Input and AI Parsing

Input Methods: Natural language (e.g. z = sin(x) * cos(y)) or descriptive text (e.g. projectile motion)

AI Dual Channel (switchable from the frontend):

ChannelProtocolDescription
anthropicAnthropic Messages APICompatible with all Anthropic protocol services (e.g. MiMo), uses tool_use to enforce structured output
openaiOpenAI Chat CompletionsCompatible with all OpenAI protocol services, uses function calling to enforce structured output

AI Output Structure:

{
  "type": "surface",
  "expression": "Math.sin(x) * Math.cos(y)",
  "variables": ["x", "y"],
  "domain": { "x": [-5, 5], "y": [-5, 5] },
  "parameters": [{ "name": "A", "label": "Amplitude", "min": 0.1, "max": 5, "step": 0.1, "default": 1 }],
  "animation": { "variable": "t", "speed": 1.0, "range": [0, 10], "loop": true },
  "explanation": "This is a binary sine-cosine product surface...",
  "latex": "z = \\sin(x) \\cdot \\cos(y)",
  "title": "sin(x)cos(y) Surface",
  "dimension": "3d"
}

Visualization Types: surface (surface) | curve (parametric curve) | parametric2d (2D parametric plot) | vectorField (vector field) | particles (particles)

2. 3D/2D Visualization Rendering

Uses Plotly.js (react-plotly.js) for rendering, no WebGL required, works in all browser environments.

TypePlotly traceDescription
surfacetype: 'surface' or 'contour'3D surface / 2D contour plot, Viridis color mapping
curvetype: 'scatter3d' or 'scatter'Parametric curve, supports 2D/3D
vectorFieldtype: 'cone'Arrow field, 6x6 grid
particlestype: 'scatter3d'Particle scatter, random initialization + displacement

Interactive Features: Mouse rotation, zoom, pan (Plotly built-in), parameter sliders update graphics in real time.

3. AI Explanation Panel

  • Formulas rendered with KaTeX (full LaTeX support)
  • AI returns Chinese explanations, covering formula meaning, physical significance, and visualization highlights
  • Supports standalone explain API calls (with context)

5. Settings and Configuration

Settings Panel (gear button) allows online modification of AI channel configuration:

FieldDescription
Anthropic API KeySupports sk- and tp- formats
Anthropic Base URLFull URL (e.g. https://xxx.com/anthropic/v1)
Anthropic ModelModel name (e.g. mimo-v2.5)
OpenAI API KeySupports sk- format
OpenAI Base URLLeave empty to use official API
OpenAI ModelLeave empty for default gpt-4o

After saving, changes are written to backend/.env and take effect immediately without restarting the backend.

Channel Status Detection: Sidebar buttons display availability status; unconfigured channels are automatically grayed out and disabled.

Model List: /api/models automatically derives the /v1/models endpoint from ANTHROPIC_BASE_URL, fetches the model list, and filters out non-text models such as TTS.

6. Loading State

During AI parsing (approximately 5-15s), the canvas area displays a loading overlay:

  • Spinning spinner
  • Sliding progress bar animation
  • Hint text “AI is parsing the formula and generating visualization…”
