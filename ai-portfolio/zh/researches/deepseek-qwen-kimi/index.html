DeepSeek、Qwen、Kimi 前沿研究报告

中国三大AI模型：MoE架构、GRPO算法、混合推理、训练效率革新

DeepSeek、Qwen、Kimi 前沿研究报告

报告日期：2026年5月30日 覆盖时间范围：2024年12月 — 2025年7月

摘要

DeepSeek（深度求索）、Qwen（通义千问，阿里巴巴）和 Kimi（月之暗面，Moonshot AI）是当前中国 AI 大模型领域最具影响力的三支研究力量。它们在 2024-2025 年间取得了一系列突破性进展，核心创新集中在以下方向：

  • 强化学习（RL）驱动的推理能力涌现：DeepSeek-R1-Zero 证明纯 RL 训练即可激发链式推理能力
  • MoE（混合专家）架构的工程化突破：以极低成本训练出 671B 参数模型
  • 混合推理模式：Qwen3 实现单模型内 /think 与 /no_think 模式的自由切换
  • 长上下文与长链推理优化：Kimi k1.5 提出 long2short 技术，将长推理链能力蒸馏到短推理模型中
  • 训练效率革新：FP8 混合精度训练、Muon 优化器等将训练成本降低一个数量级

一、DeepSeek（深度求索）

1.1 DeepSeek-V3：极致效率的 MoE 架构

发布时间：2024年12月

架构核心：

  • 总参数量：671B（6710亿）
  • 每 token 激活参数：37B（约370亿）
  • 架构类型：Mixture-of-Experts（MoE），采用细粒度专家路由

关键技术创新：

技术说明
Multi-head Latent Attention (MLA)将 KV Cache 压缩至低秩潜空间，推理时动态重建，KV Cache 内存减少约 93%，同时保持或超越标准 MHA 性能，速度快于 GQA
DeepSeekMoE细粒度专家划分 + 共享专家机制，辅以无辅助损失（auxiliary-loss-free）负载均衡策略
FP8 混合精度训练在大规模训练中采用 FP8 精度，显著降低显存占用和计算成本

训练效率：

  • 训练数据：14.8 万亿 token
  • 训练成本：约 550 万美元（远低于同级别闭源模型的数亿美元量级）
  • 性能对标 GPT-4o 和 Claude 3.5 Sonnet

论文：DeepSeek-V3 Technical Report（arXiv, 2024年12月）

1.2 DeepSeek-R1：RL 驱动的推理革命

发布时间：2025年1月

DeepSeek-R1-Zero：纯 RL 训练的推理涌现

这是 2025 年最具震撼力的 AI 研究发现之一：

  • 训练方式：仅使用强化学习（RL），完全不使用监督微调（SFT） 作为前置步骤
  • 核心发现：模型在 RL 训练过程中自发涌现出链式推理（Chain-of-Thought）、自我反思和自我纠错能力
  • “Aha Moment”（顿悟时刻）：训练过程中模型突然开始生成更长、更结构化的推理链，展现出”学会思考”的涌现行为
  • 意义：挑战了”SFT 是构建推理模型必要前提”的传统假设，证明大语言模型可以通过纯试错强化学习发展出复杂的问题求解策略

局限性：R1-Zero 存在可读性差、语言混杂等问题，因此最终的 R1 模型加入了冷启动 SFT 阶段以改善输出质量。

DeepSeek-R1 完整训练流水线

冷启动 SFT → RL (GRPO) → 拒绝采样 SFT → 第二阶段 RL

GRPO（Group Relative Policy Optimization）

GRPO 是 DeepSeek 提出的核心 RL 算法创新：

工作原理：

  1. 对每个 prompt 生成一组响应（如 64 个候选）
  2. 使用基于规则的奖励函数（如数学/代码正确性检查）对每个响应评分
  3. 奖励在组内相对归一化：advantage = (reward - mean(rewards)) / std(rewards)
  4. 使用 PPO 风格的裁剪目标更新策略

关键优势：

  • 去除 critic/value 模型：用组内平均奖励替代价值函数，大幅减少显存和计算开销
  • 比标准 PPO 更加计算高效

二、Qwen（通义千问，阿里巴巴）

2.1 Qwen2.5 系列：专业化模型矩阵

发布时间：2024年9月起陆续发布

模型规模：0.5B — 72B 参数，覆盖从边缘设备到云端的全场景需求

模型系列参数规模特化方向
Qwen2.50.5B / 1.5B / 3B / 7B / 14B / 32B / 72B通用语言能力
Qwen2.5-Coder1.5B / 7B / 14B / 32B代码生成与编程任务
Qwen2.5-Math1.5B / 7B / 72B数学推理（支持中英双语）

技术规格：

  • 上下文窗口：最高 128K token
  • 训练数据：最高 18 万亿 token
  • 架构：基于 Transformer，改进注意力机制和训练稳定性

基准表现：

  • Qwen2.5-Coder 在 HumanEval、MBPP 等代码基准上与 DeepSeek-Coder、GPT-4 Turbo 竞争
  • Qwen2.5-Math 在 GSM8K、MATH 等数学基准上表现优异

2.2 QwQ-32B：推理模型探索

发布时间：2024年底 / 2025年初

  • 定位：类似 OpenAI o1 的推理型模型
  • 参数规模：32B
  • 训练方法：在强大的预训练基础模型上应用强化学习（RL）
  • 基准表现：在 AIME 2024（数学竞赛）、LiveCodeBench（代码）、GPQA（研究生水平科学问答）等推理基准上，接近或达到 DeepSeek-R1 和 o1-mini 的水平
  • 开源：开放权重发布

技术报告声明：

“这些发现凸显了 RL 应用于在大量世界知识上预训练的稳健基础模型时的有效性，并表明扩展 RL 有潜力将模型性能提升到超越传统预训练和后训练方法的水平。“

核心创新：混合推理模式（Hybrid Thinking Mode）

Qwen3 最具标志性的创新是单模型内的推理模式切换：

  • /think 模式：模型执行扩展的链式推理（类似 o1 风格），适合复杂数学、逻辑、编程任务
  • /no_think 模式：模型直接给出答案，适合快速响应场景
  • 思考预算控制：用户可根据任务复杂度选择何时启用深度推理

意义：一个模型同时服务推理和非推理两种使用场景，无需部署不同模型。

基准表现

  • 旗舰模型 Qwen3-235B-A22B 在多个基准上与 DeepSeek-R1、OpenAI o1、GPT-4o、Grok-3 竞争或超越
  • Qwen3-30B-A3B（MoE）以更少的激活参数超越了 QwQ-32B
  • 在数学推理（AIME、MATH-500）、代码生成（LiveCodeBench）、通用知识（GPQA）、多语言任务（支持 100+ 语言）上均有强劲表现

开源贡献

Qwen3 全系列以开源权重发布，可在 Hugging Face 和 ModelScope 获取，提供基座和指令微调版本。

三、Kimi（月之暗面，Moonshot AI）

3.1 长上下文先驱

Kimi 是中国 AI 领域长上下文处理的先驱：

  • 最早支持 200K token 上下文窗口的商业模型之一
  • 后续将上下文窗口扩展至数百万 token 级别
  • 在长文档理解、多轮对话保持、跨文档推理等场景建立优势
  • 在 KV Cache 优化和长上下文训练方面进行了大量工程创新

3.2 Kimi k1.5：RL 推理的长链到短链蒸馏

发布时间：2025年1月

核心定位：多模态推理模型，基于强化学习训练

关键技术贡献

Long2Short（长链到短链）技术：

  • 将长链推理（Long-CoT）能力蒸馏到短链推理（Short-CoT）模型中
  • 在保持强推理能力的同时，显著减少推理时的 token 消耗和延迟
  • 使模型在实际部署中更加高效

其他技术要点：

  • 长-CoT 到短-CoT 的蒸馏
  • RL 训练中的部分展开（Partial Rollout）策略
  • 改进的奖励建模
  • 课程学习（Curriculum Learning）提升训练效率

多模态能力

k1.5 设计为处理多模态输入（文本和图像），能够跨模态进行推理。

基准表现

在数学、代码和推理基准上取得竞争性结果，在多个任务上与 OpenAI o1 等领先推理模型持平或超越。

论文：Kimi k1.5: Scaling Reinforcement Learning with LLMs（Moonshot AI, 2025年1月）

3.3 Kimi k2：万亿参数 MoE + 新型优化器

发布时间：2025年7月

Kimi k2 是 Moonshot AI 的最新旗舰模型，代表了其在架构和训练方法上的最新突破。

模型规格

属性数值
总参数量约 1 万亿（1T）
每 token 激活参数约 32B
架构Mixture-of-Experts (MoE)
训练数据15.5 万亿 token
许可证Modified MIT License（开源）

核心创新：Muon 优化器

Kimi k2 的一个关键训练创新是采用了 Muon 优化器（由 Keller Jordan 等人提出，论文标题：Muon: An Optimizer for Neural Networks with Orthogonalizing Momentum）：

  • 核心思想：在动量缓冲区上施加谱正交化（Spectral Orthogonalization）
  • 优势：相比传统 Adam 优化器，以更少的训练步数达到更强的性能
  • 意义：首次在万亿参数级别模型的训练中成功应用新型优化器

能力定位

  • 代码生成：在 LiveCodeBench、SWE-bench 等代码基准上表现强劲
  • 数学推理：在 MATH 等数学基准上具有竞争力
  • 智能体/工具使用：专为自主智能体工作流设计，支持复杂的工具调用和任务编排
  • 开源：以开放权重形式发布

4.3 关键技术趋势

趋势一：RL 成为推理能力的核心驱动力

三家机构不约而同地将强化学习作为提升推理能力的关键手段：

  • DeepSeek 证明了纯 RL 可以涌现推理能力（R1-Zero）
  • Qwen 在预训练基础模型上应用 RL 获得显著性能提升（QwQ）
  • Kimi 通过 RL 训练实现长链推理并蒸馏至短链模型（k1.5）

趋势二：MoE 架构成为效率标杆

三家均采用或支持 MoE 架构：

  • 通过稀疏激活大幅降低推理成本
  • 在保持甚至超越密集模型性能的同时，将激活参数控制在 20-40B 量级
  • 使万亿参数级模型的训练和部署成为现实

趋势三：推理效率的精细化控制

从”只能深度推理”走向”按需推理”：

  • Qwen3 的 /think / /no_think 模式切换
  • Kimi k1.5 的 Long2Short 蒸馏
  • DeepSeek R1 的多规格蒸馏模型系列

趋势四：训练成本的指数级下降

  • DeepSeek-V3 以约 550 万美元训练出对标 GPT-4o 的模型
  • FP8 精度、GRPO 等技术创新大幅降低计算需求
  • MoE 架构进一步提升训练效率

六、结论

DeepSeek、Qwen 和 Kimi 三家机构在 2024-2025 年间的研究成果，共同勾勒出中国 AI 大模型领域的前沿图景：

  1. DeepSeek 以工程效率和理论创新见长——MLA、GRPO、R1-Zero 的纯 RL 推理涌现，每一项都是对现有范式的挑战
  2. Qwen 以产品化和生态广度取胜——从 0.6B 到 235B 的全尺寸覆盖、Qwen3 的混合推理模式、以及最大的开源模型矩阵
  3. Kimi 以长上下文和训练方法论创新为特色——Long2Short 技术、万亿参数 MoE 训练、Muon 优化器的工程验证

三家共同推动了几个关键趋势的加速：RL 驱动推理、MoE 架构普及、训练成本下降、以及推理效率的精细化控制。这些进展不仅在技术层面具有重要意义，也在全球 AI 竞争格局中产生了深远影响。
