DeepSeek Codex TUI

终端AI编程助手，Python Textual + Rich TUI，6个工具，4×2正交模式系统

DeepSeek Codex TUI v1.0 — 功能完整报告

一、项目概述

DeepSeek Codex TUI 是一个类似 OpenAI Codex CLI 的终端 AI 编程助手，使用 DeepSeek API 作为后端（OpenAI 协议兼容）。核心能力：AI 通过 Tool Calls 读写文件、执行命令、搜索代码，在终端中交互式完成编程任务。

技术栈

组件技术
TUI 框架Python Textual + Rich
HTTP 客户端httpx（异步流式）
API 协议OpenAI 兼容（DeepSeek / OpenAI / 任何兼容 API）
Key 存储keyring（OS 级安全存储）+ 文件回退
沙箱Docker 容器（可选）/ 本地 subprocess

启动方式

cd D:/claude/agenttui
pip install -e .
dcodex

二、核心功能

2.1 流式聊天

  • 实时流式输出 AI 回复（Markdown 渲染）
  • 思考过程（reasoning_content）可折叠显示
  • 处理时间显示（“Sautéed for 3s”）
  • Token 用量和缓存命中实时更新

2.2 Tool Calls（工具调用）

AI 可调用 6 个工具完成编程任务：

工具功能参数
read_file读取文件内容path, start_line, end_line
write_file写入/覆盖文件path, content
edit_file查找替换编辑path, old_text, new_text
run_command执行终端命令command, cwd
search_code正则搜索代码pattern, path, file_filter
list_files列出目录结构path

2.3 工具调用展示（Claude Code 风格）

    2 │ import math
    3 │
    4 │ def add(a, b):
    5 │     return a + b
    ... +269 lines (ctrl+o to expand)
  ✓ auto

● Bash(python calculator.py)

  ✓ done

三、4×2 正交模式系统

3.1 Runtime 模式（AI 在做什么）

模式说明切换
autoAI 按权限模式正常执行工具/plan 退出时
planAI 只规划，不执行任何工具/plan 进入时

3.2 Permission 模式（工具怎么被批准）

模式文件读文件写命令执行安全等级
suggest确认确认确认★★★★★
auto-edit自动自动确认★★★☆☆
full-auto自动自动自动★★☆☆☆
bypass自动自动自动（⚠️ 危险）★☆☆☆☆

3.3 审批弹窗（Claude Code 风格）

● Write(calc.py)
  write_file(path='calc.py', content='...')
  Do you want to proceed?

  > 1. Yes
    2. Yes, and always allow for this session
    3. No
  • Yes — 本次允许
  • Yes, and always allow — 会话内同类工具自动通过
  • No — 拒绝，AI 收到 “用户拒绝了此操作”

四、/ 命令系统

命令功能
/help显示所有可用命令
/clear清空当前对话
/new新建会话（选择工作目录）
/settings打开设置界面
/plan进入/退出规划模式

/model切换模型（支持 --1m 1M 上下文）

/effort设置思考深度（low/medium/high/xhigh/max）
/balance查询账户余额
/cost显示本次会话 token 用量
/compact压缩对话上下文
/system查看/设置系统提示词
/history查看会话历史

4.1 /plan 规划模式工作流

● /plan
📋 规划模式已启用 (runtime: plan)
权限模式不变: suggest

● 我想做个计算器
→ AI 问澄清问题（不执行工具）

● 科学计算器，Python
→ AI 输出实施方案

● /plan
✓ 退出规划模式 → auto
当前权限: suggest

4.2 /model —1m 1M 上下文

/model deepseek-v4-pro --1m    →  deepseek-v4-pro[1m]
/model deepseek-v4-flash --1m  →  deepseek-v4-flash[1m]

八、安全特性

8.1 Docker 沙箱

  • Docker 可用时自动使用容器执行命令
  • python:3.12-slim 镜像
  • --network none 禁用网络
  • 工作目录挂载到 /workspace
  • 退出时自动清理容器
  • 不可用时回退到本地 subprocess

8.2 危险命令检测

DANGEROUS_COMMANDS = {
    "rm", "rmdir", "del", "rd", "format", "mkfs",
    "shutdown", "reboot", "halt", "poweroff",
    "sudo", "su", "runas",
    "chmod", "chown", "icacls",
    "kill", "killall", "taskkill",
}

8.3 API Key 安全存储

  • 优先使用 OS keyring（Windows Credential Locker / macOS Keychain / Linux SecretService）
  • 回退到加密文件存储
  • 不硬编码在代码中

九、性能优化

优化项实现
Markdown 更新限频每 0.08 秒最多更新一次
思考过程更新限频每 0.3 秒更新一次
滚动限频每 0.15 秒 scroll_end 一次
参数过滤inspect.signature 自动过滤 AI 返回的多余参数
会话持久化conversations.json 本地存储

十、错误处理

场景处理
网络连接失败Toast 通知 + 重试
401 认证失败跳转设置页
429 限速显示等待时间
流中断（无 [DONE]）保留已生成内容
工具执行超时30 秒自动终止
命令执行超时Docker 容器自动清理
ESC 中断保留已生成内容 + 清理 agent 消息

十三、已知限制

  1. Docker 依赖：沙箱功能需要 Docker Desktop 运行
  2. 无 undo：文件写入后无法自动撤销
  3. 无测试：未包含自动化测试
  4. 单用户：不支持多用户并发
  5. 离线不可用：需要网络连接到 API
