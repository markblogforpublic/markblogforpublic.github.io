Phos v1.0

本地小模型工具包，llama-cpp-python + Streamlit GUI，完全离线运行

Phos v1.0 — 功能报告

项目概述

Phos 是一个基于 llama-cpp-python 的本地小模型工具包，通过 Streamlit GUI 提供可视化操作界面。用户可以加载本地 GGUF 格式模型，完成翻译、摘要、聊天、分类、抽取、视觉理解、语音合成等任务，全部在本地运行，无需联网。

系统架构

minimalLLMagent/
├── app.py               # Streamlit GUI 入口（侧边栏 + 7 个 Tab）
├── requirements.txt     # 依赖声明
├── phos/
│   ├── __init__.py      # 包导出
│   ├── model.py         # ModelManager — 核心模型管理
│   ├── tasks.py         # 文本任务函数（5 个）
│   ├── vision.py        # 图片理解任务
│   ├── tts.py           # Orpheus TTS 语音合成
│   └── presets.py       # 参数预设持久化
└── models/              # GGUF 模型存放目录

核心模块详解

1. phos/model.py — 模型管理核心

ModelManager 是整个系统的中心类，通过 llama-cpp-python 直接加载 GGUF 模型。

主要方法：

方法说明
load(path, n_ctx, n_gpu_layers)加载纯文本模型
load_multimodal(path, mmproj, n_ctx, n_gpu_layers)加载多模态模型（带 mmproj 视觉投影器）
unload()释放模型内存
complete(messages, temperature, max_tokens)单轮生成，返回完整文本
stream(messages, temperature, max_tokens)逐 token 流式生成，yield (text, stats)
_has_images(messages)检测消息中是否包含图片

特性：

  • 启动时自动扫描 NVIDIA CUDA DLL 路径，支持 GPU 加速
  • 流式生成时自动检测多模态消息，回退到非流式模式以保证图片正常处理
  • 每次生成后记录性能统计（总耗时、token 数、tokens/sec、首 token 延迟）

2. phos/tasks.py — 文本任务

五个独立函数，均接收 manager 参数：

函数功能输入输出
translate(text, target_lang, source_lang)翻译文本 + 目标语言str
summarize(text, style, max_length)摘要/改写文本 + 风格参数str
classify(text, categories)文本分类文本 + 类别列表dict（category, confidence, reasoning）
extract(text, fields)信息抽取文本 + 字段列表dict（字段名→值）
chat(messages, system_prompt)多轮对话消息历史str

classify/extract 使用正则从输出中提取 JSON，不依赖 json_mode，兼容所有模型。

3. phos/vision.py — 图片理解

函数功能
describe_image(image_path, prompt)图片描述，支持自定义 prompt
ask_about_image(image_path, question)图片问答

图片文件读取后 base64 编码，通过 image_url 类型消息传入模型。

4. phos/tts.py — 语音合成

基于 Orpheus-3B TTS 模型 + SNAC 音频解码器：

文本 → Orpheus GGUF 模型 → 音频 token → SNAC 解码器 → PCM 音频 → WAV 文件
组件作用
synthesize(text, voice, temperature)主入口，返回 WAV bytes
pcm_to_wav(pcm_bytes, sample_rate)PCM 转 WAV
SNAC (24kHz)音频解码器，自动下载缓存

支持 8 种音色：tara, leah, jess, leo, dan, mia, zac, zoe

5. phos/presets.py — 参数预设

函数功能
load_presets()读取 ~/.phos/presets.json
save_preset(name, settings)保存 n_ctx, n_gpu_layers, temperature, max_tokens
delete_preset(name)删除预设

GUI 功能（app.py）

侧边栏

功能说明
目录选择弹出 Windows 原生目录选择器，递归扫描 .gguf 文件
模型类型三种：文本模型 / 多模态视觉（需 mmproj）/ 语音合成
n_ctx上下文长度（1024~131072，默认 4096）
计算模式GPU (CUDA) 或 CPU
GPU 层数选择 GPU 时可设（默认 99，全部卸载）
加载/卸载按钮操作，显示当前模型状态

主区域 Tab

Tab 1：聊天

  • 流式输出（逐字显示）
  • 维护多轮对话历史
  • 可选 System prompt
  • 多模态模型时：显示图片上传接口（📎 附加图片），图片随消息一起发送
  • 性能统计条：总耗时、Token 数、Tokens/s、首 Token 延迟

Tab 2：翻译

  • 源语言/目标语言下拉框（支持自动检测）
  • 目标语言默认中文

Tab 3：摘要/改写

  • 风格选择：concise / formal / casual / bullet-points / elaborate
  • 自定义输出长度

Tab 4：分类/抽取

  • 分类模式：输入文本 + 类别列表 → JSON（category, confidence, reasoning）
  • 抽取模式：输入文本 + 字段列表 → JSON（字段→值）

Tab 5：视觉（多模态模型加载时显示）

  • 上传图片 + 自定义问题
  • 返回模型分析结果

Tab 6：语音合成

  • 输入文字 + 选择音色
  • 播放音频 + 下载 WAV
  • 依赖 snac 包

启动方式

cd minimalLLMagent
pip install -r requirements.txt
streamlit run app.py

浏览器访问 http://localhost:8501

技术特点

  1. 零配置自动检测：启动时自动扫描 CUDA DLL，无需手动设置环境变量
  2. GPU/CPU 一键切换：侧边栏 radio 切换，n_gpu_layers 自动调整
  3. 多模态自动适配：加载带 mmproj 的模型后，聊天 Tab 自动出现图片上传接口
  4. 流式生成：聊天 Tab 逐字显示，普通任务非流式保证稳定性
  5. 性能可视化：每次生成后显示 tokens/sec、TTFT 等关键指标
  6. 预设持久化：参数组合保存到 ~/.phos/presets.json，重启不丢失
  7. 原生目录选择器：使用 tkinter 弹出系统级目录选择对话框
