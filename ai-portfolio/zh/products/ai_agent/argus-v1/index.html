一、项目概述

Argus 是一个 AI 驱动的桌面自动化代理，通过截图→AI 分析→执行操作的循环，让 AI 模型能够像人一样操控电脑。支持多供应商 API（Anthropic 官方 / OpenAI 兼容第三方），内置安全分析、输入法自动切换、智能滚动等功能。

核心架构

用户输入任务
    ↓
┌─────────────────────┐
│  截屏 (mss)          │ ← 动态分辨率检测
│  编码为 base64 PNG    │
└─────────┬───────────┘
          ↓
┌─────────────────────────┐
│  发送截图 + 工具定义      │ ← 自动选择 Anthropic/OpenAI 协议
│  via provider.send()     │
└─────────┬───────────────┘
          ↓
   stop_reason?
   ├─ "tool_use" → 安全分析 → 弹窗确认(dangerous) → PyAutoGUI 执行
   └─ "end_turn" → 输出结果 → 结束

3.3 智能输入法切换

语言检测 (_detect_language)：

  • 基于 Unicode 范围检测 CJK 字符（中日韩统一表意文字、假名、韩文等）
  • 阈值：CJK 字符占比 > 15% 即判定为 CJK 文本
  • 混合文本（如 “Hello你好”）也会触发切换

输入法切换 (_ensure_ime_for_text)：

  • 检测到 CJK 文本 → 自动发送 Shift 键切换输入法
  • 适用于微软拼音、搜狗输入法等默认用 Shift 切换中英文的输入法
  • 切换后通过剪贴板粘贴输入（Ctrl+V）

Windows 键映射：

用户输入映射到 PyAutoGUI

3.4 安全分析系统

三级安全评估：

级别触发条件处理方式
SAFEmouse_move, screenshot, cursor_position, wait自动执行
CAUTIONleft_click, right_click, type, key, scroll, drag日志记录，自动执行
DANGEROUS特定快捷键、危险命令、连续快速点击弹窗确认（tkinter messagebox）

DANGEROUS 触发规则：

类别触发条件
危险快捷键alt+f4, ctrl+alt+del, ctrl+shift+esc, delete, backspace, alt+tab, win+d, win+l, win+r, ctrl+w, ctrl+q, f4
危险命令rm , del , format , shutdown, restart, taskkill, sudo , chmod 777, rd /s, diskpart, bcdedit
连续点击同类点击动作超过 5 次

弹窗确认：

  • 使用 tkinter.messagebox.askyesno
  • 显示动作详情 + 风险描述
  • 用户点”是”执行，点”否”跳过并通知 AI 尝试其他方案

两种模式：

  • safe（默认）— DANGEROUS 动作需确认
  • dangerous — 所有动作自动执行

六、运行方式

# CLI 交互模式
python computer_use.py

# 直接测试（跳过 CLI 交互）
python test_run.py

CLI 流程：

  1. 自动检测屏幕分辨率
  2. 从 .env + cli_config.json 自动导入 API 配置
  3. 显示已有 profiles，选择或新建
  4. 检测模型能力（vision / computer_use / tool_use）
  5. 选择安全模式（safe / dangerous）
  6. 输入任务，开始执行
