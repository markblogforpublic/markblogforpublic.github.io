Astrognosy v2 - 功能报告

项目概述

Astrognosy 是一款类 OpenAI Deep Research 的多模型深度研究工具。用户输入研究问题后，系统自动进行多轮搜索、阅读、分析，最终生成带引用的 Markdown 格式深度研究报告。

核心差异化：支持所有兼容 OpenAI API 格式的模型提供商（OpenAI、DeepSeek、Moonshot、Qwen、本地 Ollama 等），开箱即用，无需绑定特定平台。

核心功能

1. 研究循环引擎 (Research Agent Loop)

系统自动执行以下循环（最多 3-5 轮迭代）：

用户输入问题
    ↓
[规划] LLM 分解问题为 3-5 个子问题 + 搜索词 + 搜索类别
    ↓
[搜索] 并行多源搜索：通用 + 学术 + GitHub + Wikipedia
    ↓
[内容提取] 三层降级策略获取网页正文
    ↓
[阅读] LLM 分析搜索结果，提取关键事实和引用
    ↓
[分析] 评估信息完整性，判断是否继续搜索
    ↓
  信息足够？──否──→ 回到 [搜索]
    ↓ 是
[写作] 生成带内联引用的 Markdown 报告

每一步通过 SSE 实时推送到前端，用户可观察完整的研究过程。

搜索策略：

  • 默认多源并发：同时在所有可用类别中搜索，结果按 URL 去重聚合
  • LLM 智能路由：规划阶段的 LLM 根据问题类型决定搜索哪些类别
  • 降级机制：Provider 失败时自动尝试同类别的其他 Provider

4. 指定网站搜索（域名过滤）

用户可在设置面板中添加关注的域名列表（如 arxiv.org、github.com、nature.com），搜索时自动应用域名限制：

  • DuckDuckGo / Brave / SearXNG：使用 site:xxx.com 前缀
  • Tavily：使用内置的 includeDomains 参数
  • GitHub：使用 org: 过滤
  • 不支持域名过滤的 Provider（Wikipedia、OpenAlex）照常搜索

5. 浏览器截图 + 多模态分析

三层降级的内容获取策略，处理 JS 重度渲染的页面：

1️⃣ Jina Reader 文本提取（快速、低成本，适合静态页面）
   ↓ 内容不足或提取失败？
2️⃣ Playwright 截图 → base64 → 多模态 LLM 分析（准确、处理动态页面）
   ↓ 模型不支持 vision？
3️⃣ Playwright DOM 文本提取（移除脚本/样式/广告后提取正文）

多模态模型自动检测：

系统自动识别当前模型是否支持视觉（Vision）能力，已知支持的模型：

厂商支持 Vision 的模型关键词
OpenAIgpt-4o, gpt-4-turbo, gpt-4-vision
Anthropicclaude-3, claude-3.5, claude-4
Googlegemini-1.5, gemini-2, gemini-pro-vision
阿里qwen-vl, qwen2-vl
其他deepseek-vl, llava, minicpm-v
  • 支持 vision → 截图方案可用，设置面板显示绿色标识
  • 不支持 vision → 仅用文本提取，设置面板显示灰色标识
  • 用户可在设置中手动开关截图功能

6. 实时进度展示

前端通过 SSE 接收研究进度，展示：

  • 阶段指示：规划 → 搜索 → 阅读 → 分析 → 写作 → 完成
  • 每步详情：搜索词、结果数量、提取状态
  • 思考过程：模型的分析和判断（如果模型支持）
  • 来源列表：已收集的所有引用来源，按类别分色显示
  • 时间戳：每个步骤的精确时间

7. 研究报告渲染

使用 react-markdown + remark-gfm 渲染最终报告：

  • 完整的 Markdown 支持（标题、列表、表格、代码块、引用）
  • 暗色主题适配
  • 内联引用链接（点击跳转原始来源）
  • 引用来源列表（编号对应内联引用）

研究引擎 Prompt 设计

规划阶段 (PLANNING_PROMPT)

将用户问题分解为 3-5 个子问题，每个子问题生成搜索词和搜索类别（web/academic/code/knowledge），以 JSON 格式输出。

阅读阶段 (READING_PROMPT)

从搜索结果中提取关键事实、引用来源、信息可信度和信息缺口，以 JSON 格式输出。

分析阶段 (ANALYSIS_PROMPT)

评估已收集信息的完整性（0-100 分），判断是否需要继续搜索，如需要则生成下一轮搜索建议。

写作阶段 (WRITING_PROMPT)

根据所有收集的信息撰写结构化的 Markdown 研究报告，包含内联引用 [数字] 和参考来源列表。

快速开始

# 1. 安装依赖
npm install

# 2. 安装 Playwright 浏览器（仅截图功能需要）
npx playwright install chromium

# 3. 配置环境变量（可选）
cp .env.example .env.local
# 编辑 .env.local 填入你的 API Keys

# 4. 启动开发服务器
npm run dev

# 5. 打开浏览器
# http://localhost:3000

已知限制

限制说明
DuckDuckGo 反爬频繁请求可能被限流，建议配置 Tavily 或 Brave 作为备选
截图性能Playwright 启动 Chromium 有一定开销，建议仅在模型支持 Vision 时启用
内存存储研究报告存储在内存中，页面刷新后丢失（后续可加数据库）
SSE 超时长时间研究（>5分钟）可能触发代理超时，需配置超时时间
arXiv 速率建议请求间隔 3 秒以上，避免被封禁
