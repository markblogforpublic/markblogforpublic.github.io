一、项目概述

Aetherion 是一个 AI 原生数据库操作系统，将数据库从被动存储转变为智能协作系统。它通过 Anthropic Claude API 实现自然语言到 SQL 的转换、数据库分析、风险评估和自主优化，同时提供严格的安全治理框架。

核心理念：

数据库不应只是存储信息。
它们应当理解、分析、优化并协作。

三、双层执行矩阵

Aetherion 的核心设计是 执行权限层 × 认知智能层 的双层矩阵，产生 9 种独立 AI 角色。

3.1 执行权限层（WHAT — AI 被允许做什么）

模式说明安全等级
plan只读模式。AI 可以分析 schema、生成 SQL、模拟执行、解释查询计划，但不能修改数据库。最高
allow_edit人机协作模式。AI 可执行查询、修改记录、优化表、创建索引，但每个关键操作需要用户确认。中等
bypass全自主执行模式。AI 可自主优化系统、执行迁移、修复性能问题。Guardian 系统仍阻止致命操作。受控

3.2 认知智能层（HOW — AI 如何思考和推理）

引擎代号映射模型特点Token 消耗
Hōkiboshi彗星Claude Opus深度系统推理，Extended Thinking，多步推理链极高
Sfumato晕涂法Claude Sonnet探索性推理，假设驱动，模糊推理，模式发现中高
Vivace活泼Claude Haiku高速执行，最小延迟，工具优先，快速查询生成低

3.3 九种角色组合

HōkiboshiSfumatoVivace

Bypass自主数据库架构师自主研究代理高频基础设施代理

四、核心模块详解

4.1 认知引擎（aetherion/ai/）

客户端（client.py）

  • 封装 Anthropic Python SDK，支持自定义 API 地址和密钥
  • Prompt Caching：系统提示和 Schema 上下文使用 ephemeral 缓存控制
  • 流式输出：stream_message() 实时返回文本块
  • Extended Thinking：Hōkiboshi 模式下启用深度推理（thinking_budget=10000）
  • Token 追踪：记录 input/output/cache_read/cache_creation tokens 和成本估算
  • 环境冲突自动清理：检测并清除 ANTHROPIC_AUTH_TOKEN 等 Claude Code 会话遗留变量

认知引擎基类（cognitive_engine.py）

  • 抽象接口：build_reasoning_chain() 和 get_tools() 由各引擎实现
  • 自动路由：根据 thinking_budget 选择 Extended Thinking / Tool Use / Simple Message 模式
  • Schema 上下文注入：通过 schema_context 参数传递数据库结构

三个引擎实现：

引擎系统提示风格工具集max_tokensthinking_budget
Hōkiboshi结构化、系统性、多步推理analyze_schema, explain_query, execute_readonly_query, analyze_indexes819210000
Sfumato叙事性、探索性、假设驱动explore_table, run_query, find_correlations, detect_anomalies40960
Vivace简洁、行动导向、工具优先execute_query, get_table_info, show_tables20480

4.2 数据库适配器（aetherion/db/）

抽象基类（base.py）

  • 统一接口：connect(), disconnect(), get_schema(), execute(), explain(), get_stats()
  • 上下文管理器：with adapter: 自动连接/断开
  • Schema 转字符串：to_context_string() 生成 AI 可读的数据库结构描述
  • 数据模型：ColumnInfo, IndexInfo, ForeignKeyInfo, TableInfo, SchemaInfo, QueryResult, ExecutionPlan, DatabaseStats

SQLite 适配器（sqlite_adapter.py）

  • WAL 模式：启用 PRAGMA journal_mode=WAL 优化并发读性能
  • 外键约束：PRAGMA foreign_keys=ON
  • 完整 Schema 提取：PRAGMA table_info, PRAGMA foreign_key_list, PRAGMA index_list
  • 行数统计：SELECT COUNT(*) 精确计数
  • EXPLAIN 支持：解析操作码统计 Seq Scan / Index Scan

PostgreSQL 适配器（postgresql_adapter.py）

  • 通过 psycopg2-binary 连接
  • information_schema 查询列、外键、索引
  • pg_indexes / pg_class / pg_stat_activity 获取统计信息
  • EXPLAIN JSON 格式输出，递归解析执行计划树

4.3 安全治理层（aetherion/security/）

AST 验证器（ast_validator.py）

所有 AI 生成的 SQL 在执行前必须通过 AST 解析验证。使用 sqlglot 库。

检查项动作说明
DROP DATABASE / DROP SCHEMABLOCKGuardian 系统永久阻止
TRUNCATE TABLEBLOCK永久阻止
DELETE 无 WHEREBLOCK必须指定条件
UPDATE 无 WHEREBLOCK必须指定条件
ALTER TABLEREQUIRE_CONFIRM需要用户确认
CREATE INDEXREQUIRE_CONFIRM可能影响写入性能
多语句注入BLOCK检测 ; 分隔的多条 SQL
JOIN 数量 > 4FLAG警告复杂查询

风险评分引擎（risk_engine.py）

三维风险评估：

  1. 风险等级：LOW → MEDIUM → HIGH → CRITICAL
  2. 影响范围：NONE → SINGLE_ROW → MULTI_ROW → TABLE → SCHEMA → DATABASE
  3. 置信度：0.0 ~ 1.0（低于阈值触发沙箱执行）

综合评分公式：score = base_risk × 0.6 + scope_factor × 0.3 + confidence_factor × 0.1

执行路径决策：

  • LOW → 所有模式直接执行
  • MEDIUM → Bypass 直接，Allow Edit 确认，Plan 阻止
  • HIGH → 沙箱执行 + 确认
  • CRITICAL → Guardian 永久阻止

角色注册表（role_registry.py）

预配置 9 种角色，每种包含：

  • 工具白名单（限制该角色可用的工具）
  • 安全宪法类型（Safety / Research / Enterprise / Speed）
  • Token 预算和 Thinking 预算
  • 流式输出开关

4.4 治理矩阵（aetherion/governance/matrix.py）

核心编排器，处理管道 8 步：

用户输入 → ① 解析角色 → ② 获取 Schema → ③ 构建上下文
         → ④ 获取引擎 → ⑤ AI 处理 → ⑥ 提取内容和工具调用
         → ⑦ 验证 SQL（AST + Risk）→ ⑧ 执行或返回

支持运行时模式切换（/mode hokiboshi, /exec bypass），权限升级时发出警告。

4.5 审计日志（aetherion/audit/logger.py）

  • 存储：SQLite append-only 数据库
  • 记录内容：时间戳、会话 ID、执行/认知模式、角色名、用户输入、生成的 SQL、风险等级、执行结果、Token 消耗
  • 查询接口：按 session_id、action、limit 过滤
  • 统计接口：总条目数、各动作计数、Token 消耗汇总

4.6 配置系统（aetherion/config.py）

优先级叠加：CLI 参数 > 环境变量 > 配置文件（aetherion.yaml）> 默认值

支持的环境变量：

  • ANTHROPIC_API_KEY — API 密钥
  • ANTHROPIC_BASE_URL — API 基础地址（支持代理/中继服务）
  • AETHERION_MODEL — 默认模型名

五、CLI 命令行（aetherion/cli.py）

aetherion chat                    # 交互式对话（默认 Vivace 模式）
aetherion chat --mode hokiboshi   # 深度推理模式
aetherion chat --mode sfumato     # 探索模式
aetherion analyze --db sqlite:x   # 分析数据库
aetherion query "自然语言问题"      # 单次查询
aetherion status                  # 显示当前配置
aetherion serve                   # 启动 Web 仪表盘 API 服务器
aetherion serve --model mimo-v2.5 # 指定模型启动
aetherion serve --port 8001       # 指定端口

交互模式内置命令：

  • /mode <hokiboshi|sfumato|vivace> — 切换认知模式
  • /exec <plan|allow_edit|bypass> — 切换执行模式
  • /usage — 显示 Token 消耗统计
  • /schema — 显示数据库结构
  • /quit — 退出

6.3 五大面板

① AI 对话面板（默认面板）

  • 实时流式输出：通过 SSE（Server-Sent Events）逐字显示 AI 回复
  • Markdown 渲染：支持代码块、表格、列表、引用块
  • 模式切换：右上角下拉选择执行模式和认知模式
  • SQL 预览：AI 生成的 SQL 以等宽代码块显示
  • 角色标识：每次回复显示使用角色名和执行时间
  • 阻断提示：被 Guardian 系统拦截时显示红色 BLOCKED 标签
  • 输入体验：Enter 发送，Shift+Enter 换行，自动聚焦

② 数据库浏览器面板

  • 左侧树形导航：表 → 列（主键金色钥匙图标，外键蓝色图标）→ 类型标签
  • 行数统计：每张表右侧显示行数
  • 查询编辑器：等宽字体 SQL 输入框，Ctrl+Enter 快速执行
  • 结果表格：固定表头，hover 行高亮，NULL 值半透明显示
  • 连接管理：顶部输入数据库 URL，支持 SQLite 和 PostgreSQL
  • 数据库统计：表数量、总行数、文件大小

③ 安全与治理面板

  • 当前模式展示：执行模式 + 认知模式下拉切换，显示当前活跃角色名和描述
  • SQL 验证器：粘贴 SQL 后一键验证，显示 AST 分析结果
    • 语句类型（SELECT/INSERT/UPDATE/DELETE/DDL）
    • 表数量、JOIN 数量、是否有 WHERE 子句
    • 违规项列表（带严重度图标）
  • 风险评估：风险等级（带颜色标签）、评分、影响范围、置信度
    • 被阻止时显示锁图标和阻止原因
  • 角色矩阵：展示全部 9 种角色，当前激活的角色高亮显示

④ 监控与审计面板

  • 统计卡片：总条目数、输入 Token、输出 Token、动作类型数
  • 动作分布：allowed / blocked / confirmed 各自的计数
  • 审计日志表：时间、模式、角色、输入、SQL、风险、动作、耗时、Token 消耗
    • 表头固定，hover 行高亮
    • 时间使用等宽字体对齐

⑤ 设置面板（模态窗口）

  • API Key：密码输入框 + 显示/隐藏切换，显示当前遮罩密钥
  • Base URL：支持自定义代理/中继地址（如 https://token-plan-cn.xiaomimimo.com/anthropic）
  • Model：支持自定义模型名（如 mimo-v2.5、claude-sonnet-4-20250514）
  • 测试连接：一键测试 API 连通性，显示连接结果和模型名
  • 持久化保存：配置写入 aetherion.yaml 文件并同时设置环境变量
  • 配置路径提示：显示当前配置文件的完整路径

6.4 侧边栏导航

  • 四个面板导航图标 + 文字标签
